# Contributing to Psychic Canary

Thanks for your interest in contributing to Psychic Canary!  We're excited to build a better market regime detection tool together.

All contributions are welcome: code improvements, bug fixes, new features, documentation, tests, or even suggestions.

## Code of Conduct

Please follow our CONDUCT.MD in all interactions.

## How to Contribute

1. **Fork the repository** and clone it locally.
2. **Create a branch** for your feature or fix: `git checkout -b your-branch-name`.
3. Make your changes. Follow existing code style (PEP 8 for Python).
4. **Test your changes** thoroughlyâ€”add or update tests if possible.
5. Commit with clear messages (e.g., "Fix phase inference bug in volatile regimes").
6. **Push to your fork** and open a Pull Request (PR) against the `main` branch.
7. Describe your changes in the PR: what it does, why it's needed, and any testing done.

## Reporting Bugs or Suggesting Features

- Search existing issues first.
- Open a new issue with:
  - A clear title and description.
  - Steps to reproduce (for bugs).
  - Expected vs. actual behavior.
  - Relevant logs or screenshots.

## Style Guidelines

- Python code: Follow PEP 8. Use black for formatting if possible.
- Commit messages: Conventional Commits style (e.g., `feat: add new phase detector`, `fix: handle API rate limits`).
- Add docstrings to new functions/classes.

## Questions?

Feel free to open an issue or discuss on the repo's Discussions tab.

Thank you for helping improve Psychic Canary!
